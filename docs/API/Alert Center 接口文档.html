# Alert Center 接口文档

# 主题路由：/alert/config/themes/

# 请求类型：GET

  • 用途：获取主题列表（含主题具体内容）

  • 请求参数：无

  • 响应字段声明

    字段名称 字段类型 是否必要 含义
    type string 是 主题的类型
    id string 是 主题的名称

    其他字段略，当前版本仅需展示主题列表即可

{
  	"code": 200,
  	"message": "OK",
    "data": [
        {
            "type": "dingding_signal_action_card",
            "buttons": [
                {
                    "title": "{{ message.title }}",
                    "actionURL": ""
                }
            ],
            "text": "{{ message.content }}",
            "id": "default-message-style.yml"
        },
        {
            "buttons": [
                {
                    "title": "{{ alert.alarmInfo.level }}",
                    "actionURL": ""
                }
            ],
            "text": "{{ alert }}",
            "type": "dingding_signal_action_card",
            "id": "default-alert-style.yml"
        }
    ]
}

# 请求类型：POST

  • 用途：创建新主题
  • 其他声明略，当前版本仅需展示主题列表即可

# 请求类型：PUT

  • 用途：更新主题
  • 其他声明略，当前版本仅需展示主题列表即可

# 请求类型：DELETE

  • 用途：删除主题
  • 其他声明略，当前版本仅需展示主题列表即可

# 主题名称路由：/alert/config/themes_names/

# 请求类型：GET

  • 用途：获取所有主题名称（含主题具体内容）

  • 请求参数：无

  • 响应字段声明

    响应字段声明

    字段名称 字段类型 是否必要 含义
    type string 是 主题所属的媒介类型
    请注意：和前面的主题类型不是同一个概念
    id string 是 主题的名称

  • 示例

    {
  	"code": 200,
  	"message": "OK",
    "data": [
        {
            "id": "default-message-style.yml",
            "type": "dingding"
        },
        {
            "id": "default-alert-style.yml",
            "type": "dingding"
        }
    ]
}

# 通知媒介路由：/alert/config/mediums/

# 请求类型：GET

  • 用途：获取通知媒介列表

  • 请求参数：无

  • 响应字段声明

    字段名称 字段类型 是否必要 含义
    json的key string 是 媒介名称
    type string 是 媒介的类型
    theme string 是 该媒介使用的主题
    url string 否 钉钉url，仅当媒介类型为时包含此参数
    secret string 否 钉钉加签验证secret，仅当媒介类型为dingding时，可能含有此参数
    SECRETID string 否 腾讯语音鉴权ID，仅当媒介类型为tx-voice时包含此参数
    SECRETKEY string 否 腾讯语音鉴权KEY，仅当媒介类型为tx-voice时包含此参数
    VOICESDKAPPID string 否 腾讯语音APPID，仅当媒介类型为tx-voice时包含此参数
    alternate-contact list 否 备用联系人列表，仅当媒介类型为tx-voice时包含此参数
    duty-contact string 否 值班表文件位置，仅当媒介类型为tx-voice时包含此参数

  • 示例

    {
    "code": 200,
  	"message": "OK",
    "data": [
        {
            "id":"bigdata-dingding",
            "type": "dingding",
            "url": "https://oapi.dingtalk.com/robot/send?access_token=xxxxx",
            "secret": "xxxxxxx",
            "theme": "dingding-card-default-style"

        },
        {
            "id":"bigdata-tx-phone",
            "type": "tx-voice",
            "SECRETID": "",
            "SECRETKEY": "",
            "VOICESDKAPPID": "",
            "theme": "default-alert-voice",
            "alternate-contact": [
                "+86xxxxxxxx"
            ]

        },
        {
            "id":"bigdata-message-dingding",
            "type": "dingding",
            "url": "https://oapi.dingtalk.com/robot/send?xxxxxxxxx",
            "secret": "xxxxxxxxxx",
            "theme": "dingding-card-default-message-style"

        }
    ]
}

# 请求类型：POST/PUT

  • 用途：创建通知媒介/更新通知媒介

  • 请求body：有

  • 请求body说明

    字段名称 字段类型 是否必要 含义
    json的key string 是 媒介名称
    type string 是 媒介的类型
    theme string 是 该媒介使用的主题
    url string 否 钉钉url，仅当媒介类型为时包含此参数
    secret string 否 钉钉加签验证secret，仅当媒介类型为dingding时，可能含有此参数
    SECRETID string 否 腾讯语音鉴权ID，仅当媒介类型为tx-voice时包含此参数
    SECRETKEY string 否 腾讯语音鉴权KEY，仅当媒介类型为tx-voice时包含此参数
    VOICESDKAPPID string 否 腾讯语音APPID，仅当媒介类型为tx-voice时包含此参数
    alternate-contact list 否 备用联系人列表，仅当媒介类型为tx-voice时包含此参数
    duty-contact string 否 值班表文件位置，仅当媒介类型为tx-voice时包含此参数

  • 请求参数示例

    // 创建/更新更新钉钉类型媒介示例
{
    "data": [
        {
            "id": "bigdata-dingding",
            "type": "dingding",
            "url": "https://oapi.dingtalk.com/robot/send?access_token=xxxxx",
            "secret": "xxxxxxx",
            "theme": "dingding-card-default-style"

        }
      ]
}

// 创建/更新腾讯电话类型示例
{
    "data": [
        {
            "id":"bigdata-tx-phone",
            "type": "tx-voice",
            "SECRETID": "",
            "SECRETKEY": "",
            "VOICESDKAPPID": "",
            "theme": "default-alert-voice",
            "alternate-contact": [
                "+86xxxxxxxx"
            ]

        }
    ]
}

  • 响应参数说明

    字段名称 字段类型 是否必要 含义
    status string 是 动作状态，仅作详细信息说明，任何非200/3xx状态码均为异常响应

  • 响应示例

    {
  "code": 200,
  "message": "OK",
  "status":"更新成功"
}  //http code 200
{
    "code": 200,
    "message": "OK",
    "status":"error xxxxxxx"
} //http code 502

# 通知组路由：/alert/config/sendgroups/

# 请求类型：GET

  • 用途：获取通知组

  • 请求参数：无

  • 响应字段声明

    字段名称 字段类型 是否必要 含义
    json的key string 是 通知组名称
    secure string 是 随机字符串，sendgroup的验证字段
    alert-medium list 是 普通告警媒介的名称
    high-alert-medium list 是 严重告警媒介的名称
    message-medium list 是 消息通知媒介的名称

  • 响应示例

    {
  	"code": 200,
  	"message": "OK",
    "data": [
        {
            "id": "bigdata",
            "secure": "xjgksangka",
            "alert-medium": [
                "bigdata-dingding"
            ],
            "high-alert-medium": [
                "bigdata-tx-phone"
            ],
            "message-medium": [
                "bigdata-message-dingding"
            ]

        }
    ]
}

# 过滤器(屏蔽)路由：/alert/config/filter/

# 请求类型：GET

  • 用途：获取过滤器列表

  • 请求参数：无

  • 响应字段声明

    字段名称 字段类型 是否必要 含义
    json的key string 是 过滤器id
    condition dict 是 条件。共三种类型：project，项目；resource，资源；trigger，触发器
    time_start string 是 开始时间
    time_end string 是 结束时间
    regular string 是 是否重复，当前版本改字段无效
    info string 是 过滤器描述
    action string 是 动作，符合过滤器条件是的动作：forbid，屏蔽；sendnow，立即发送
    sendto string 是 将该条过滤器的表更发送至哪个通知组，通知组需从上述接口获取
    creater string 是 创建者，根据op系统当前登陆用户生成

  • 响应示例

    {
    "code": 200,
 	"message": "OK",
    "data": [
        {
            "id": "123",
            "condition": {
                "project": "gt-test",
                "resource": "gt-host1",
                "trigger": "cpu使用率"
            },
            "time_start": "2021/12/27 09:30",
            "time_end": "2021/12/27 14:00",
            "regular": "no",
            "creater": "tester",
            "info": "this a test",
            "action": "forbid",
            "sendto": "bigdata
        }
    ]
}

# 请求类型：POST/PUT

  • 用途：创建或更新过滤器

  • 响应字段声明
