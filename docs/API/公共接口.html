# 数据交互格式

# 前端请求

类型 格式
非文件 raw json
文件 form-data
#### 后端响应
{
code: int,
message: string,
data: list, #[{}, {}, ]
}
字段 类型
------- ------
code int
message string
data list
##### code：状态码
code值 含义
------ -------------------------
200 成功
405 请求方法错误
401 用户名/密码错误/token失效
597 批量操作中部分报错
598 展示报错信息
599 服务端数据出错

# 路由访问

# 请求头

Authorization，值为 jwt+空格+token

# token

  • 来源：用户登陆后分配
  • 过期时间：默认1天

# 路由访问要求

除了登陆路由(/login/)外，所有路由都需要携带上述指定请求头信息

# 响应

#异常
{
    "data": [],
    "code": 403,
    "message": "您没有执行该操作的权限。"
}
#正常：返回执行页面内容

# 登陆特点

单点登陆：用户每次登陆后，都会生成最新的token，之前持有的token将会失效

  • 防止用户被恶意登陆
  • 清除token，解决jwt无法删除token的bug
