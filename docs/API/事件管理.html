# 事件类型

# 路由

查询方法：get

请求内容

响应内容

字段名称 字段类型 含义
id int 自增id
etype int 事件类型，请参考公共数据
name string 名称
comment string 备注

    "data": [
        {
            "id": 1,
            "etype": 0,
            "name": "服务端",
            "comment": ""
        },
        {
            "id": 2,
            "etype": 1,

            "comment": "用户范围"
        },
        {
            "id": 3,
            "etype": 3,
            "name": "游戏-合服维护",
            "comment": ""
        }
    ],
    "code": 200,
    "message": "OK"
}

# 变更报表

# 路由

/event/groupby/

# 获取变更报表内容

查询方法：get

注意：值班统计中，不传递部门时，默认统计所有部门，没有传递时间时，默认统计当月

如/fault/groupby/?gbkey=uptype&start=2021-05-20&end=2021-06-01

字段名称 字段类型 含义
start date 开始时间，格式yy-mm-dd
end date 结束时间，格式yy-mm-dd
gbkey string 分组key
name string 业务系统
sertype string 按照业务系统同时更新对象分组
uptype string 按照业务系统同时维护类型分组

响应内容

字段名称 字段类型 含义

seconds string 秒数字符串
name__name string 业务系统名称
uptype__name string 更新对象名称
sertype__name string 维护类型名称

# IT日常变更

# 路由

/event/itchange/

#etype类型，内容从原型来
0:设备采购
1：设备维修

# 获取IT日常变更内容

查询方法：get

响应内容：

字段名称 字段类型 含义
id int 自增id
etype int 事件类型，由前端写死，默认值为0
comment string 事件内容
asset list 固产管理页面中id列表，[id, ]
invite string 事件申请人
deal string 处理人
begin date 处理日期，格式yyyy-mm-dd
end date 归还日期，格式yyyy-mm-dd
{
    "data": {
        "asset": [
            {
                "id": 3,
                "code": "apple-110"
            },
            {
                "id": 4,
                "code": "saytesatewe"
            }
        ],
        "event": [
            {
                "id": 7,
                "etype": "设备采购",
                "comment": "kg",
                "invite": "技术中心",
                "deal": "技术中心",
                "begin": "2015-11-02",
                "end": "2016-12-02",
                "asset": [
                    3,
                    4
                ]
            }
        ]
    },
    "code": 200,
    "message": "OK"
}

# 导出IT日常变更内容

# 路由

/event/itchange-export/

  "data": [id1, id2]
